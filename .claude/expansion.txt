1. Overview
This spec defines the expanded Business Profile structure in the Content Library. The expansion serves three purposes:
Enable multi-industry selection (max 2) for businesses that operate across sectors
Capture services and products with integrated pricing for Chatbot responses
Ensure day-one readiness for Chatbot and Design Visualiser tools
The Business Profile becomes mandatory for first-time users before any tools can be activated. This is enforced at the Dashboard level.

3.1.1 Industry Removal Warning Modal
When a user clicks to deselect an industry, a modal warning appears using the .perm-modal-overlay / .perm-modal pattern.
Modal content:
Heading: &quot;Remove [Industry Name]?&quot;
Warning text explaining consequences
List of industry-specific tools owned for that industry (if any)
List of tools using industry context: CB (system prompt), DV (render types), ID (news sources), SP (industry analysis)
Note: &quot;Existing tool outputs will not be deleted but may reference the removed industry&quot;
Buttons: Cancel (re-selects the chip) / Confirm Removal (leaves deselected)

5.2 Placeholder Text Bug Fix
Current issue: BP fields with placeholder/guidance text are being pulled into SP as actual content.
Fix required:
SP prefill must check for empty string or null values. Placeholder text must remain in the placeholder attribute only and never be stored as field values. SP displays empty field if BP value is empty, not placeholder text.
6. Chatbot Integration
The expanded Business Profile provides day-one knowledge for the Chatbot without requiring manual FAQ entry.
6.1 CB Knowledge Sources
The Chatbot system prompt draws from:
Business Profile (all panels)
Content Library items tagged CB with status Approved
Tool Outputs in CL tagged CB
6.2 Pricing Disclosure Setting
CB Settings (not BP) includes a pricing disclosure control:
Share actual prices (uses price data from BP)
Share price ranges only
Don&apos;t discuss pricing — direct customers to request a quote
7. Design Visualiser Integration
DV render types are restricted based on the user&apos;s selected industries. Users who select multiple industries (max 2) have access to render types from all selected industries.
Render types per industry are defined in Industry Taxonomy Spec v1.2.
8. Mandatory BP Completion
⚠️ Business Profile completion is mandatory before any tools can be activated. This is enforced at the Dashboard level.
On first login, users are directed to complete BP before accessing tools. The Dashboard displays a completion prompt until all mandatory fields are populated.
Mandatory fields are marked &quot;Yes&quot; in the Required column in Section 3. Total: 25 mandatory fields, 5 optional fields.
9. &quot;Other&quot; Option UI Component
Fields with &quot;Other&quot; options (Services, Products, Licences, Service Area) use a consistent add/remove UI pattern.
Behaviour:
When user types a custom entry and clicks Add, the entry appears as a .filter-pill with an × remove button in the same row as the standard options. This is not a separate component — custom entries are rendered alongside standard pills.
Uses .filter-pill[data-custom] with inline × span for remove button.
