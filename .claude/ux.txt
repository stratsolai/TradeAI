1. Overview
This specification defines UX improvements for the Business Profile (BP) panels in the Content Library. These improvements address issues identified during user testing of the BP completion flow.

1.1 Issues Identified
Services/Products/Licences chip lists are unorganised — not grouped, not alphabetical
Address entry requires manual typing — no autocomplete
Price range input fields are too narrow and have poor default behaviour
Marketing Theme wizard asks duplicate questions already answered in other BP panels
Colour picker requires hex code knowledge — SME owners do not know hex codes

1.2 Scope
This spec covers changes to:
Panel 3 — Services
Panel 4 — Products
Panel 5 — Credentials &amp; Support (Licences section)
Panel 2 — Location &amp; Contact (Address autocomplete)
Panel 6 — Marketing Theme (Wizard redesign, Colour picker)

2. Grouped Chip Layout
Services, Products, and Licences chip lists must be reorganised into grouped, alphabetised, collapsible sections.

2.1 Current State
All chips displayed in a single flat list
No grouping by category
No alphabetical ordering
Difficult to scan and find relevant items

2.2 Required Changes
2.2.1 Grouping Structure
The current bp-industry-data.js has flat lists per industry with no sub-categories. Sub-groups must be CREATED in bp-industry-data.js before the UI can display grouped chips.

Code must:
Propose a logical sub-group structure for services, products, and licences
Get owner approval for the proposed groupings
Update bp-industry-data.js with the new category structure
Then implement the grouped UI

2.2.2 Collapsible Sections
Once sub-groups exist, each category group has:
A clickable heading showing the category name
A chevron icon indicating expand/collapse state
Collapsed by default — user clicks to expand and see chips
When expanded, chips are displayed below the heading
Selected chips remain visible (with count) even when section is collapsed

2.2.3 Alphabetical Order
Within each category group, chips are sorted alphabetically A–Z.

2.2.4 Visual Pattern
Use the existing accordion/expandable pattern from staxai-auth.css. Code must find and use these existing styles. If Code cannot find them, STOP and ask — they exist.

3. Address Autocomplete
The address entry fields in Panel 2 (Location &amp; Contact) must use autocomplete to reduce manual typing and ensure valid Australian addresses.

3.1 Current State
Street Address, Suburb, State, Postcode are separate text inputs
User must type each field manually
No validation of address accuracy

3.2 Required Changes
3.2.1 Autocomplete Integration
Integrate Google Places Autocomplete API (or equivalent) for the Street Address field.
User starts typing in Street Address field
Dropdown shows matching Australian addresses
User selects an address from dropdown
Street, Suburb, State, and Postcode fields auto-populate from selection
User can still manually edit any field after auto-population

3.2.2 API Considerations
Restrict to Australian addresses only
Use session tokens to minimise API costs
Graceful fallback if API unavailable — manual entry still works

4. Price Range Input Fields
The pricing fields for Services and Products need improvements for usability.

4.1 Current Issues
Input fields too narrow for realistic price figures (e.g. $1,200,000)
Default value of 0 remains when user clicks to enter — requires manual deletion
No number formatting

4.2 Required Changes
Widen input fields to accommodate 7+ digit figures
Clear the field on focus if value is 0 — user can type immediately
Format as numbers only — no $ symbol in the field (label indicates currency)
Add thousand separators on blur for readability (e.g. 1,200,000)
Strip formatting on focus for easy editing

5. Marketing Theme Wizard Redesign
The Marketing Theme wizard (Panel 6) has significant UX issues that require a redesign.

5.1 Current Issues
5.1.1 Duplicate Questions
The wizard asks questions that have already been answered in earlier BP panels. Upon first login, users complete the entire BP in one flow — they should not be asked the same information twice.

Any code that currently uses data from these wizard questions must be updated to pull from the source panels instead (Panel 1, Panel 3, Panel 5).

5.1.2 Colour Picker
The current colour picker requires users to enter hex codes (e.g. #4A6D8C). SME business owners typically do not know hex codes for their brand colours.

5.2 Required Changes
5.2.1 Remove Duplicate Questions
Remove the following questions from the Marketing Theme wizard entirely:
Years in Business — already captured in Panel 1
Qualifications and Certifications — already captured in Panel 5
Which services do you specialise in — already captured in Panel 3

The AI generation that uses this data must pull from the source panels:
Years in Business → Read from profiles.years_in_business
Licences/Certifications → Read from profiles.licences
Services → Read from profiles.services

5.2.2 Redesigned Colour Picker
Replace the hex input with a visual colour selection interface:

Preset Palette (recommended approach):
Display a grid of preset colour swatches (12–16 common brand colours)
User clicks a swatch to select
Include common colours: blues, greens, reds, oranges, purples, greys, black
&quot;Custom colour&quot; button opens a visual colour picker for advanced users
Selected colour shown as a large preview swatch
Hex code displayed below for reference (read-only unless &quot;Custom&quot; selected)
